# DevOps Documentation: FBOT (MotionHunter SaaS)

Данный документ предназначен для инженеров, отвечающих за развертывание, масштабирование и обслуживание системы MotionHunter.

## Архитектура системы

Система построена на микросервисной архитектуре и упакована в Docker-контейнеры:

- **bot**: Python-приложение (Telethon + Aiohttp). Занимается мониторингом Telegram-каналов, обработкой AI и API для Web App.
- **db**: Реляционная СУБД PostgreSQL 15.

## Требования

- Docker Engine 20.10+
- Docker Compose V2
- Минимум 1 ГБ ОЗУ (рекомендуется 2 ГБ для стабильной работы OpenAI и множества сессий)

## Конфигурация (.env)

Файл конфигурации должен находиться в папке `./app/.env`.

| Переменная          | Описание                                       | Обязательно |
| :------------------ | :--------------------------------------------- | :---------- |
| `BOT_TOKEN`         | Токен главного управляющего бота от @BotFather | Да          |
| `TELEGRAM_API_ID`   | API ID с my.telegram.org                       | Да          |
| `TELEGRAM_API_HASH` | API Hash с my.telegram.org                     | Да          |
| `OPENAI_API_KEY`    | Ключ для фильтрации лидов нейросетью           | Да          |
| `WEBAPP_URL`        | Публичный URL вашего приложения (домен или IP) | Да          |
| `DB_HOST`           | Хост базы данных (в Docker-compose это `db`)   | Да          |
| `DB_PASS`           | Пароль к PostgreSQL                            | Да          |

## Развертывание (Deployment)

### 1. Подготовка окружения

Склонируйте репозиторий и настройте `.env`:

```bash
cp app/.env.example app/.env
nano app/.env
```

### 2. Запуск системы

Запуск в фоновом режиме с принудительной пересборкой образа:

```bash
docker-compose up --build -d
```

### 3. Проверка статуса

```bash
docker-compose ps
docker-compose logs -f bot
```

## Обслуживание и Persistence

### Хранение данных (Volumes)

В `docker-compose.yml` настроены постоянные хранилища:

- `pgdata`: Данные PostgreSQL. Не удаляйте этот volume без необходимости.
- `./app/.sessions`: Файлы сессий Telegram (\*.session). **Критически важны** для работы юзерботов. При потере этого каталога всем пользователям придется логиниться заново.

### Бэкап базы данных

Рекомендуется настроить cron-задачу для создания дампов:

```bash
docker exec motionhunter-db pg_dump -U postgres motionhunter > backup_$(date +%F).sql
```

### Мониторинг логов

Логи бота пишутся в stdout контейнера. Для глубокого аудита используется файл:

- `./app/data/audit.jsonl` — детальные логи всех срабатываний (кто, кому, когда ответил).

## Troubleshooting

### Бот не запускается / ошибка Ngrok

Если в логах ошибка Ngrok — убедитесь, что вы удалили упоминания Ngrok из кода (в текущей версии он вырезан и заменен на `WEBAPP_URL`).

### Ошибка "Database is locked" или "Connection refused"

PostgreSQL имеет Healthcheck. Бот не запустится, пока база не будет готова принимать соединения. Если база не стартует — проверьте права на папку с данными на хосте.

### FloodWaitError

Если юзерботы получают бан от Telegram — увеличьте переменные `MIN_DELAY` и `MAX_DELAY` в `.env` для имитации более естественного поведения человека.
